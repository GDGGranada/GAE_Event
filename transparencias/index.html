Cloud Computing

Introducción

Intro a Cloud Computing

  • Cloud Computing: paradigma que permite ofrecer servicios de computación a través de Internet.
  • Los servicios se clasifican en tres grandes niveles:
    • IaaS(Infrastructure as a Service)
    • PaaS (Platform as a Service)
    • SaaS (Software as a Service)

Intro a Cloud Computing

Clasificación de los servicios de Cloud Computing

¿Que es PaaS?

  • Una Plataforma como servicio (platform as a service, PaaS) es:
    • la encapsulación de una abstracción de un ambiente de desarrollo y,
    • el empaquetamiento de una serie de módulos o complementos que proporcionan una funcionalidad horizontal (persistencia de datos, autenticación, mensajería, etc.).
  • Ejemplos comerciales: Google App Engine, Force.com, Windows Azure...

PaaS: Ventajas

Permite al cliente múltiples ventajas, entre otras:

  • Escalabilidad garantizada
  • Pago por consumo = Inversión progresiva
  • Desarrollo más sencillo.
  • Integración con el resto de la plataforma
  • Administración remota
  • Despliegue transparente.
  • Altísima disponibilidad del 99.99%.

PaaS: Inconvenientes

  • Al integrarnos con un plataforma tenemos ciertas limitaciones y estamos supeditados a un proveedor, para lo bueno y para lo malo:
    • Duración del contrato.
    • Licencias de software.
    • Problemas de movilidad.
  • Herramientas muy limitadas
  • Surge: Plataforma Abierta como Servicio (Open Platform as a Service)
    • Ejemplo: OpenShift de Red Hat

Google Cloud Platform

Google App Engine

¿Que es GAE?

  • Google App Engine permite ejecutar aplicaciones web en la infraestructura de Google
  • Características:
    • Las aplicaciones son fáciles de crear, de mantener y de ampliar al ir aumentando el tráfico y las necesidades de almacenamiento de datos.
    • Es posible tener un dominio propio para la URL de nuestra webapp o bien podemos usar un subdominio appspot.com

¿Que es GAE?

Características

  • Admite aplicaciones escritas en varios lenguajes de programación:
    • Java
    • Python
    • Go - ¡¡Experimental!!
  • Pago por uso y sin coste alguno: hasta 500 MB de almacenamiento y suficiente CPU y ancho de banda como para permitir un servicio eficaz de alrededor de cinco millones de visitas a la página al mes.
  • Es posible subir y ofrecer una aplicación a la tienda de aplicaciones “Google Apps Marketplace”.

Restricciones:

  • Permiso de lectura a los archivos del sistema de archivos.
  • Ejecuta código a través de consultas HTTP
  • Las webapps de Java tienen acceso a clases seguras del JRE estándar. (Listado completo)
  • Las webapps de Python tienen acceso a las APIs seguras de la biblioteca Python estándar, sin uso de módulos en C, admite bibliotecas Python de terceros. (Listado completo)
  • Las sesiones y los servicios tienen un tiempo de respuesta
  • Las webapps no pueden crear nuevos hilos de ejecución

App Engine webapp framework

Al igual que Django o Rails, webapp es una implementación del patrón de diseño MVC (Model View Controller).

Una aplicación web que use el framework WSGI compilant webapp de Google sería la siguiente:

from google.appengine.ext import webapp
from google.appengine.ext.webapp.util import run_wsgi_app

class MainHandler(webapp.RequestHandler):
    def get(self):
        self.response.headers["Content-Type"] = "text/plain"
        self.response.out.write("Hello World!")

application = webapp.WSGIApplication([('/', MainHandler)],
                                            debug=True)
def main():
    run_wsgi_app(application)

if __name__ == "__main__":
    main()

Memcache

  • Memoria caché distribuida con utilidad de almacenar datos en ella en lugar del DataStore
  • Se conserva el mayor tiempo posible
  • Los datos pueden ser expulsados de la memoria al introducir nuevos datos y la memoria es baja
  • Valor máximo almacenado: 1 MB

from google.appengine.api import users
from google.appengine.ext import webapp
from google.appengine.ext.webapp.util import run_wsgi_app

class MainPage(webapp.RequestHandler):
    def get(self):
        user = users.get_current_user()

        if user:
            self.response.headers['Content-Type'] = 'text/plain'
            self.response.out.write('Hola, ' + user.nickname())
        else:
            self.redirect(users.create_login_url(self.request.uri))

application = webapp.WSGIApplication(
                                     [('/', MainPage)],
                                     debug=True)

def main():
    run_wsgi_app(application)

if __name__ == "__main__":
    main()

Uso de las plantillas

Sustituye el controlador MainPage del fichero saludos/saludos.py por el siguiente código:

class MainPage(webapp.RequestHandler):
    def get(self):
        greetings_query = Greeting.all().order('-date')
        greetings = greetings_query.fetch(10)

        if users.get_current_user():
            url = users.create_logout_url(self.request.uri)
            url_linktext = 'Salir'
        else:
            url = users.create_login_url(self.request.uri)
            url_linktext = 'Acceder'

        template_values = {
            'greetings': greetings,
            'url': url,
            'url_linktext': url_linktext,
            }

        path = os.path.join(os.path.dirname(__file__), 'index.html')
        self.response.out.write(template.render(path, template_values))

Subida de webapp

Registro de una webapp

  • Puedes crear y administrar aplicaciones web App Engine desde la consola de administración de App Engine en la siguiente URL:
    • https://appengine.google.com/
  • Los usuarios de Google App Engine Launcher pueden acceder a esta URL haciendo clic en el botón "Panel de control".
  • Introduce: nombre de usuario y contraseña de Google cuando el sistema lo solicite
  • Para crear una nueva aplicación, haz clic en el botón para crear una aplicación y sigue las instrucciones para registrar el ID de la aplicación.
  • Edita el archivo app.yaml y, a continuación, cambia el valor de application: por el ID de la aplicación que has registrado.

Subida de webapp

Subida de una webapp

  • Para subir a Google App Engine la webapp que has creado, ejecuta: 
    • gdggranada@dev ~$ python appcfg.py update saludos/
  • O bien haz clic en "Implementar" en Google App Engine Launcher.
  • Introduce: nombre de usuario y contraseña de Google cuando el sistema lo solicite
  • Si has configurado un nombre de dominio gratuito appspot.com, la URL de tu sitio web empezará con el ID de la webapp:
    • http://application-id.appspot.com
